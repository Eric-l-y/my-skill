# 代码审查检查清单

对当前变更进行系统化审查，逐项检查以下内容。

## 审查维度

### 1. 正确性
- 逻辑是否正确，边界条件是否处理
- 错误处理是否完善（try/catch、null 检查）
- 并发安全性（竞态条件、死锁）

### 2. 可读性
- 命名是否清晰（变量、函数、类）
- 函数是否单一职责，长度是否合理
- 注释是否必要且准确（不写废话注释）

### 3. 性能
- 是否有不必要的循环或重复计算
- 数据库查询是否有 N+1 问题
- 大数据量场景是否考虑分页/流式处理

### 4. 安全
- 用户输入是否校验和转义
- 敏感信息是否硬编码（密钥、密码）
- SQL 注入、XSS 等常见漏洞

### 5. 测试
- 关键路径是否有测试覆盖
- 边界条件是否测试到
- 测试是否独立、可重复

## 输出格式

按严重程度分类输出：
- 🔴 **必须修复** — 会导致 bug 或安全问题
- 🟡 **建议修改** — 影响可维护性或性能
- 🟢 **可选优化** — 代码风格或锦上添花

每个发现包含：文件位置、问题描述、修复建议。
